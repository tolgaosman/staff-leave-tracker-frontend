## Project state

This is an early-stage scaffold (`izin-takip-projesi` — Turkish for a leave/time-off tracking app) generated by `create-next-app` with a shadcn/ui base added. `app/page.tsx` has been replaced with a placeholder dashboard landing page (Turkish copy, no interactivity), and `app/layout.tsx` has custom metadata/title and a branded favicon (`public/assets/browserLogo.png`). There is still only the one route — no additional application routes, data layer, or auth exist yet.

## Commands

- `npm run dev` — start dev server (via `next dev`)
- `npm run build` — production build
- `npm run start` — run production build
- `npm run lint` — ESLint (flat config in `eslint.config.mjs`, extends `eslint-config-next`)

There is no test runner configured in `package.json` yet.

## Stack specifics that differ from typical/older Next.js knowledge

- **Next.js 16.2.10 + React 19.2** — a newer major than most training data covers. Per `AGENTS.md`, check `node_modules/next/dist/docs/` for the exact-version docs before relying on remembered Next.js APIs. Notable v16 change: request middleware is now the `proxy.ts` file convention (not `middleware.ts`).
- **shadcn/ui is configured on `@base-ui/react`, not Radix.** See `components.json` (`style: "base-nova"`) and `components/ui/button.tsx`, which wraps `@base-ui/react/button` primitives with `class-variance-authority` variants. Any new shadcn component added to this project will follow the same base-ui pattern, not the Radix-based API from older shadcn examples.
- Tailwind v4 (`@tailwindcss/postcss`, no `tailwind.config.*` file — config lives in `app/globals.css` via `@theme`/CSS variables, referenced from `components.json`).
- Icon library is `lucide-react`.
- Path alias `@/*` → repo root (`tsconfig.json`), with shadcn aliases for `@/components`, `@/lib`, `@/components/ui`, `@/hooks` (`components.json`).

## Architecture

- `app/` — App Router routes and root layout/globals.
- `components/ui/` — shadcn-generated primitives (currently just `button.tsx`); add new shadcn components here via the shadcn CLI, don't hand-roll ones that already exist upstream.
- `lib/utils.ts` — the `cn()` classname helper (`clsx` + `tailwind-merge`), the standard shadcn utility import used across `components/ui/*`.
